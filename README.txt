Информационная система «Платежный календарь»

Разработка выполнена в рамках финального этапа олимпиады (2023 год). Проект представляет собой конфигурацию на платформе «1С:Предприятие 8.3», предназначенную для оперативного планирования денежных средств, контроля лимитов и согласования платежей.

Основной упор в работе сделан на создание интерактивного автоматизированного рабочего места (АРМ) «Платежный календарь» с динамическим выводом данных и гибким управлением документами.

🛠 Технологический стек и механизмы
Платформа:1С:Предприятие 8.3 (управляемые формы)
Архитектурные объекты: 
-Документы (Заявка на расход, Планирование расхода, Планирование поступления)
-Регистры накопления (Взаиморасчеты — для учета остатков и оборотов ДС)
-Бизнес-процессы и Задачи (Согласование платежа — для маршрутизации заявок)
-Ролевая модель доступа (Сотрудник отдела, Финансовый директор)
-Механизмы компоновки и вывода данных (Табличный документ, динамическое оформление)

🚀 Функциональные возможности

1. Ролевая модель и безопасность
Ограничение прав доступа на уровне ролей.
Сотрудник отдела: создание, редактирование только своих документов и отправка их на согласование.
Финансовый директор: полный доступ + права на согласование заявок на оплату.

2. АРМ «Платежный календарь»
Интерактивная таблица, построенная на базе Табличного документа, которая позволяет:
Видеть оперативную картину движения ДС (Остаток на начало, Приход, Расход, Остаток на конец) в разрезе контрагентов, договоров и документов-регистраторов.
Строить иерархическое дерево данных.
Вводить новые документы планирования непосредственно из ячеек календаря через контекстное меню.
Изменять даты планирования документов путем их интерактивного переноса (Drag-and-Drop) между колонками-датами (с проверкой прав и статуса согласования).

3. Контроль остатков и цветовая индикация
Автоматический контроль отрицательных остатков (расход не должен превышать доступный остаток и планируемый приход).
 Визуальное выделение цветом:
   Серый — новые заявки на расход.
   Синий — заявки, по которым запущен бизнес-процесс согласования.
   Зеленый — утвержденные планируемые расходы и поступления.
   Красный — отрицательные остатки (кассовые разрывы).

4. Бизнес-процесс «Согласование платежей»
Прямо из АРМ реализовано пакетное формирование бизнес-процессов согласования для несогласованных заявок.
При подтверждении согласования автоматически формируется документ Планирование расхода.

💻 Технические особенности реализации

Вывод платежного календаря реализован программно в модуле обработки. Данные собираются с помощью запросов к виртуальным таблицам регистра накопления `Взаиморасчеты`.
